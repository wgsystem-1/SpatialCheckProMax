## 2. 검수 파일 선택

### 2.1 단일 파일 열기

**[이미지 삽입 위치: 파일 열기 메뉴]**

1. 상단 메뉴에서 `파일` > `열기`를 클릭합니다.
2. 또는 단축키 `Ctrl + O`를 사용합니다.
3. 파일 선택 대화상자에서 `.gdb` 폴더를 선택합니다.

**주의사항**:
- File Geodatabase는 **폴더 형태**입니다.
- `.gdb` 확장자가 붙은 **폴더**를 선택해야 합니다.

### 2.2 폴더 일괄 선택

**[이미지 삽입 위치: 폴더 선택 대화상자]**

여러 개의 GDB를 한 번에 검수하려면:

1. `파일` > `폴더 열기`를 클릭합니다.
2. GDB 파일들이 있는 상위 폴더를 선택합니다.
3. 검색된 GDB 목록에서 검수할 파일을 선택합니다.

```
예시 폴더 구조:
검수대상폴더/
├── 데이터셋A.gdb/      ✅ 자동 검색됨
├── 데이터셋B.gdb/      ✅ 자동 검색됨
├── 백업.gdb/           ✅ 자동 검색됨
└── 기타파일.txt        (무시됨)
```

### 2.3 드래그 앤 드롭

**[이미지 삽입 위치: 드래그앤드롭 화면]**

1. Windows 탐색기에서 `.gdb` 폴더를 선택합니다.
2. 프로그램 창으로 드래그하여 놓습니다.
3. 파일이 검수 목록에 추가됩니다.

## 3. 검수 설정

### 3.1 검수 설정 화면 열기

**[이미지 삽입 위치: 검수 설정 버튼 및 화면]**

1. 상단 메뉴에서 `설정` > `검수 설정`을 클릭합니다.
2. 또는 메인 화면의 `검수 설정` 버튼을 클릭합니다.

### 3.2 검수 단계 선택

**[이미지 삽입 위치: 검수 단계 체크박스]**

| 단계 | 검수 내용 | 권장 |
|------|----------|------|
| ☑ 1단계 | 테이블 검수 | 항상 실행 |
| ☑ 2단계 | 스키마 검수 | 항상 실행 |
| ☑ 3단계 | 지오메트리 검수 | 선택적 (시간 소요) |
| ☑ 4단계 | 속성 검수 | 권장 |
| ☑ 5단계 | 관계 검수 | 선택적 (시간 소요) |

**팁**: 
- 빠른 검수가 필요한 경우 1~2단계만 선택
- 정밀 검수가 필요한 경우 모든 단계 선택

### 3.3 검수 옵션 설정

**[이미지 삽입 위치: 검수 옵션 화면]**

| 옵션 | 설명 | 기본값 |
|------|------|--------|
| 병렬 처리 | 멀티코어 활용하여 검수 속도 향상 | ☑ 활성화 |
| QC FGDB 생성 | 검수 오류를 FileGDB로 저장 | ☑ 활성화 |
| 보고서 자동 생성 | 검수 완료 후 보고서 자동 생성 | ☑ 활성화 |
| 결과 저장 경로 | 검수 결과 파일 저장 위치 | 원본 파일 경로 |

### 3.4 결과 저장 경로 지정

**[이미지 삽입 위치: 저장 경로 설정]**

1. `찾아보기...` 버튼을 클릭합니다.
2. 결과를 저장할 폴더를 선택합니다.
3. `확인` 버튼을 클릭합니다.

```
저장되는 파일:
[지정경로]/
├── QC_원본파일명.gdb/     # QC 오류 FileGDB
├── 검수보고서_날짜.pdf     # PDF 보고서
├── 검수보고서_날짜.html    # HTML 보고서
└── 검수로그_날짜.log       # 로그 파일
```

## 4. 검수 실행

### 4.1 검수 시작

**[이미지 삽입 위치: 검수 시작 버튼]**

1. 검수할 파일을 선택합니다.
2. `검수 시작` 버튼을 클릭합니다.
3. 검수가 시작되며 진행률이 표시됩니다.

### 4.2 진행 상황 확인

**[이미지 삽입 위치: 검수 진행 화면]**

검수 진행 중 다음 정보가 표시됩니다:

| 항목 | 설명 |
|------|------|
| 전체 진행률 | 전체 검수 진행 백분율 |
| 현재 단계 | 현재 실행 중인 검수 단계 |
| 현재 작업 | 처리 중인 테이블/규칙 |
| 경과 시간 | 검수 시작 후 경과 시간 |
| 예상 잔여 시간 | 완료까지 예상 시간 |

```
예시:
╔═══════════════════════════════════════════════════╗
║  검수 진행 중...                                    ║
╠═══════════════════════════════════════════════════╣
║  전체 진행률: ████████████░░░░░░░░ 60%             ║
║  현재 단계: 3단계 - 지오메트리 검수                  ║
║  현재 작업: tn_buld (건물) - 겹침 검사              ║
║  경과 시간: 00:02:35                               ║
║  예상 잔여: 00:01:45                               ║
╚═══════════════════════════════════════════════════╝
```

### 4.3 검수 일시정지/취소

**[이미지 삽입 위치: 일시정지/취소 버튼]**

| 버튼 | 기능 |
|------|------|
| ⏸ 일시정지 | 검수를 일시 중단 (나중에 계속 가능) |
| ⏹ 취소 | 검수를 완전히 중단 |

**주의**: 취소 시 현재까지의 결과는 저장되지 않습니다.

## 6. 결과 필터링 및 검색

### 6.1 오류 필터링

**[이미지 삽입 위치: 필터 패널]**

결과 목록 상단의 필터를 사용하여 원하는 오류만 표시할 수 있습니다:

| 필터 | 설명 |

**[이미지 삽입 위치: 검색창]**

1. 검색창에 키워드를 입력합니다.
2. `Enter` 키를 누르거나 검색 버튼을 클릭합니다.
3. 키워드가 포함된 오류만 표시됩니다.

검색 가능 항목:
- 테이블명
- 오류 코드

- 세 번 클릭: 정렬 해제

## 8. 외부 GIS 연동

### 8.1 QGIS에서 오류 확인

**[이미지 삽입 위치: QGIS 연동 화면]**

1. QGIS를 실행합니다.
2. 원본 GDB와 QC_xxx.gdb를 동시에 엽니다.
3. `QC_Errors_Point`, `QC_Errors_Line`, `QC_Errors_Polygon` 레이어를 추가합니다.
4. 오류 레이어의 속성 테이블에서 `TableId`, `SourceOID`를 확인합니다.
5. 원본 레이어에서 해당 OBJECTID를 선택합니다.

```
QGIS 표현식으로 오류 객체 선택:
"OBJECTID" = 12345
```

### 8.2 ArcGIS에서 오류 확인

**[이미지 삽입 위치: ArcGIS Pro 연동 화면]**

1. ArcGIS Pro에서 원본 GDB를 엽니다.
2. QC_xxx.gdb를 추가합니다.
3. QC_Errors 레이어를 원본 레이어와 함께 표시합니다.
4. 오류 위치에서 원본 객체를 확인합니다.

### 8.3 좌표로 이동하기

검수 결과에서 오류의 X, Y 좌표를 확인하여 GIS 프로그램에서 해당 위치로 이동할 수 있습니다:

1. 오류 행을 선택합니다.
2. 마우스 우클릭 > `좌표 복사`를 선택합니다.
3. GIS 프로그램의 "좌표로 이동" 기능 사용합니다.

## 9. 검수 규칙 관리

### 9.1 규칙 비활성화하기

특정 규칙을 일시적으로 비활성화하려면:

**방법 1: GUI에서 비활성화**

1. `설정` > `검수 규칙 관리`를 클릭합니다.
2. 비활성화할 규칙의 체크박스를 해제합니다.
3. `저장` 버튼을 클릭합니다.

**방법 2: CSV 파일 직접 수정**

1. `Config/` 폴더의 CSV 파일을 텍스트 편집기로 엽니다.
2. 비활성화할 규칙의 `RuleId` 앞에 `#`을 추가합니다.
3. 또는 `Enabled` 컬럼을 `N`으로 변경합니다.

```csv
# 주석 처리로 비활성화
#LOG_TOP_REL_016,Y,PolygonContainsObjects,...

# Enabled 컬럼으로 비활성화
LOG_TOP_REL_017,N,LineWithinPolygon,...
```

### 9.2 새 규칙 추가하기

**[이미지 삽입 위치: 규칙 추가 화면]**

새로운 속성 검수 규칙을 추가하려면:

1. `Config/4_attribute_check.csv` 파일을 엽니다.
2. 기존 규칙을 참고하여 새 행을 추가합니다.

```csv
예시: 건물구분이 BDG001이면 층수가 1 이상이어야 함
LOG_DOM_ATR_NEW,Y,tn_buld,건물,bldg_nofl,IfCodeThenGreaterThanOrEqual,bldg_se;BDG001;BLDG_NOFL;1,주택은 최소 1층
```

**[이미지 삽입 위치: SHP 변환 대화상자]**

FileGDB를 Shapefile로 변환하는 기능입니다:

4. 출력 폴더를 지정합니다.
5. `변환 시작` 버튼을 클릭합니다.

### 10.2 좌표계 변환

**[이미지 삽입 위치: 좌표계 변환 대화상자]**

1. `도구` > `좌표계 변환`을 클릭합니다.
2. 원본 파일을 선택합니다.
3. 원본 좌표계를 확인합니다.
4. 대상 좌표계를 선택합니다 (예: EPSG:5179 → EPSG:4326)
5. `변환` 버튼을 클릭합니다.

지원 좌표계:
- EPSG:5179 (Korea 2000 / Unified CS)
- EPSG:5186 (Korea 2000 / Central Belt 2010)
- EPSG:4326 (WGS 84)
- EPSG:3857 (Web Mercator)

로그 레벨 필터:
- ℹ️ Information: 일반 진행 정보
- ⚠️ Warning: 경고 (작업은 계속됨)
- ❌ Error: 오류 (작업 실패)

## 12. AI 자동 수정 사용법 (v2.0 신규)

### 12.1 AI 자동 수정이란?

AI 자동 수정은 검수에서 발견된 지오메트리 오류를 인공지능 모델이 자동으로 분석하고 수정하는 기능입니다. Graph Neural Network(GNN) 기반의 모델이 지오메트리의 좌표를 분석하여 최적의 보정 값을 계산합니다.

### 12.2 지원 오류 유형

| 오류 유형 | 지원 | 설명 |
|----------|------|------|
| Self-Intersection | ✅ | 폴리곤/라인의 자체 꼬임 |
| Overlap | ✅ | 객체 간 겹침 |
| Gap | ✅ | 객체 간 틈 |
| Sliver Polygon | ✅ | 좁고 긴 슬리버 영역 |
| Invalid Geometry | ✅ | 유효하지 않은 지오메트리 |

### 12.3 AI 자동 수정 사용 방법

**[이미지 삽입 위치: AI 자동 수정 버튼]**

#### 단계별 가이드

1. **검수 실행**: 먼저 검수를 완료합니다.

2. **지오메트리 검수 탭 이동**: 검수 결과에서 지오메트리 검수 탭을 선택합니다.

3. **오류 선택**: 수정할 오류 항목을 선택합니다.
   - 단일 선택: 해당 오류만 수정
   - 전체 선택: 해당 테이블의 모든 오류 수정

4. **AI 자동 수정 버튼 클릭**:
   ```
   ╔═══════════════════════════════════════╗
   ║  [AI 자동 수정]  [GDB에 저장]         ║
   ╚═══════════════════════════════════════╝
   ```

5. **수정 결과 확인**:
   ```
   ╔═══════════════════════════════════════════════════════════╗
   ║                     AI 수정 결과                          ║
   ╠═══════════════════════════════════════════════════════════╣
   ║  처리된 오류: 15건                                        ║
   ║  성공: 12건                                               ║
   ║  실패: 3건                                                ║
   ║                                                           ║
   ║  [상세 보기]                                              ║
   ╚═══════════════════════════════════════════════════════════╝
   ```

6. **GDB에 저장**: 수정된 지오메트리를 원본 GDB에 저장합니다.

### 12.4 수정 결과 해석

| 결과 | 의미 |
|------|------|
| ✅ 성공 | AI 수정이 적용되어 유효한 지오메트리로 변환됨 |
| ⚠️ 폴백 적용 | AI 수정 실패, Buffer(0) 폴백으로 수정됨 |
| ❌ 실패 | 수정 불가 (수동 수정 필요) |

### 12.5 주의사항

1. **원본 백업**: AI 수정 전 원본 GDB를 백업해 두세요.

2. **검증 필요**: AI 수정 결과는 반드시 GIS 프로그램에서 시각적으로 확인하세요.

3. **정점 수 제한**: 정점이 500개를 초과하는 지오메트리는 AI 수정이 제한됩니다.

4. **Multi 지오메트리**: MultiPolygon, MultiLineString 등은 단일 지오메트리인 경우에만 지원됩니다.

## 13. 자주 묻는 질문 (FAQ)

### Q1. 검수 시간이 너무 오래 걸립니다.

**A**: 다음을 시도해 보세요:
1. 병렬 처리 옵션 활성화
2. 필요한 검수 단계만 선택 (3, 5단계는 시간 소요 많음)
3. GDB 파일을 SSD에 저장
4. 불필요한 검수 규칙 비활성화

### Q2. 특정 테이블만 검수하고 싶습니다.

**A**: 
1. `Config/1_table_check.csv`에서 해당 테이블만 활성화
2. 다른 테이블은 `#`으로 주석 처리

### Q3. 커스텀 속성 검사 규칙을 추가하고 싶습니다.

**A**: 
1. `Config/4_attribute_check.csv` 파일 열기
2. 새 규칙 행 추가
3. 프로그램 재시작 (설정 자동 로드)

자세한 내용은 `Config/README.md` 파일을 참조하세요.

### Q4. 검수 결과를 Excel로 내보낼 수 있나요?

**A**: 
1. 결과 탭에서 `내보내기` > `CSV 내보내기` 클릭
2. CSV 파일을 Excel에서 열기

### Q5. QC FGDB가 생성되지 않습니다.

**A**: 
1. 검수 설정에서 "QC FGDB 생성" 옵션 확인
2. 저장 경로에 쓰기 권한 확인
3. 디스크 공간 확인

### Q6. GDAL 초기화 오류가 발생합니다.

**A**:
1. `runtimes/win-x64/native/` 폴더 확인
2. Visual C++ Redistributable 2019+ 설치
3. 관리자 권한으로 실행

### Q7. AI 자동 수정이 작동하지 않습니다. (v2.0)

**A**:
1. `Resources/Models/geometry_corrector.onnx` 파일 존재 확인
2. `appsettings.json`에서 `EnableAiCorrection: true` 확인
3. 로그 파일에서 AI 모델 로드 오류 확인

### Q8. AI 수정 후 지오메트리가 이상하게 변합니다. (v2.0)

**A**:
1. AI 수정 결과는 반드시 GIS에서 시각적으로 확인하세요
2. Buffer(0) 폴백이 적용된 경우 로그에서 확인 가능
3. 문제가 지속되면 수동 수정을 권장합니다

## 14. 워크플로우 예시

### 14.1 일일 정기 검수

```
1. 프로그램 실행
      ↓
2. 검수 대상 폴더 선택 (당일 납품 데이터)
      ↓
3. 검수 설정 확인 (전체 단계 활성화)
      ↓
4. 검수 시작
      ↓
5. 결과 확인 (대시보드)
      ↓
6. 오류 있는 경우
   └── 오류 목록 확인
   └── QC FGDB를 QGIS/ArcGIS에서 열어 위치 확인
   └── 원본 데이터 수정
      ↓
7. 재검수 (수정된 파일만)
      ↓
8. 보고서 생성 (PDF)
      ↓
9. 성과물 제출
```

```
1. 프로그램 실행
      ↓
2. 상위 폴더 선택 (여러 GDB 포함)
      ↓
3. 검수 설정 조정
   └── 병렬 처리: 활성화
   └── 검수 단계: 1~2단계만 (빠른 사전검수)
      ↓
4. 일괄 검수 시작
      ↓
5. 결과 확인
   └── 문제 파일 식별
   └── 상세 검수 필요 파일 선별
      ↓
6. 선별 파일만 전체 단계 검수
      ↓
7. 최종 보고서 생성
```

```
      ↓

      ↓

      ↓

      ↓

      ↓

```
